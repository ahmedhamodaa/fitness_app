# FitTrack — Project Details

## Overview

**FitTrack** is a cross-platform fitness companion app built with **Flutter**. It presents branded as **FitTrack** in the UI (`MaterialApp` title and splash) while the Dart package name remains `fitness_app`. The app combines **Firebase Authentication** and **Cloud Firestore** for user accounts and profile data, and uses **in-app dummy data** (via `DatabaseService`) for workouts, exercises, and sample user progress—so workout content is not loaded from Firestore.

**Version:** `1.0.0+1` (from `pubspec.yaml`)  
**Dart SDK constraint:** `^3.7.0-209.1.beta`

## Features (Behavioral Summary)

1. **Splash** — Shows app logo and name, then replaces route with **Login** (does not auto-route by Firebase session).
2. **Authentication** — Email/password via Firebase Auth; **Google Sign-In**; registration writes user fields to Firestore `users` document.
3. **Home** — Welcome copy, **ProgressChart** (weekly-style line chart with static sample points), top three recommended workouts from `DatabaseService`, pull-to-refresh style via app bar refresh.
4. **Workouts tab** — Full list of dummy workouts; tap opens **WorkoutScreen** with network images and exercise list.
5. **Exercise detail** — Image, chips (muscle group, equipment), sets/reps/rest; button to launch **video URL** with `url_launcher`.
6. **Profile** — Loads **dummy** user from `DatabaseService` for BMI/history UI; parallel load of **real** Firestore `users/{uid}` for display fields; profile image pick → **Cloudinary** upload → Firestore `profileImgUrl` update; Google sign-out support.
7. **ProgressScreen** — Standalone UI with linear percent indicator and calendar; **not** currently used as a route from `HomeScreen` navigation.

## Data & Architecture Notes

- **Firestore** stores user registration/profile data (e.g. under `users` collection). Workout and exercise catalogs are **not** synced from Firestore; they come from `DatabaseService` hardcoded samples.
- **Models** (`Workout`, `Exercise`, `User`, `WorkoutHistory`, `ExerciseLog`) support `fromMap` / `toMap` for JSON-like maps (suitable for Firestore or APIs).
- **Theming** — Green-focused palette (`Constants`: primary `#4CAF50`, secondary `#388E3C`, etc.); `ThemeData` in `theme.dart` wires app bar, buttons, cards, inputs, bottom navigation, chips.
- **Fonts** — **Poppins** (Regular, Medium 500, Bold 700) from `assets/font/` per `pubspec.yaml`.
- **Assets** — `assets/images/` registered in `pubspec.yaml` (e.g. `logo.jpg`, `login_bg.jpg`, defaults for user/workout/exercise in `Constants`).

## External Services

- **Firebase** — Project configured via `firebase_options.dart` (Android, iOS, and other platforms as generated).
- **Cloudinary** — Image upload from profile uses HTTPS multipart POST to `api.cloudinary.com` with cloud name and unsigned upload preset defined in `profile_screen.dart`.

## How to Run (Typical)

1. Install Flutter matching the SDK constraint.
2. Run `flutter pub get`.
3. Ensure Firebase is configured for your targets (`google-services` / `GoogleService-Info.plist` etc., consistent with `firebase_options.dart`).
4. `flutter run` for a connected device or emulator.
